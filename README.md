# 李尔（Lear）

**一个完全由个人长期开发、使用纯Python实现的高级聊天系统，未使用任何第三方库和API等。**

李尔没有调用任何语言模型。由语料库、对话树、知识图谱、会话状态、记忆功能、角色与情绪、知识推理、实用工具、互动游戏和内容生成等近百个模块共同组成，目标是在只有Python的本地环境中探索编程式聊天机器人的能力边界，打造一个在聊天机器人领域内拥有顶尖智能性的系统。

> **“双鱼座”是李尔的前辈。** 更早开发的聊天系统，现已改名李尔。李尔在各个领域已经显著超越了前代的聊天系统。
>
> 当前仓库主要用于项目展示与持续开发。源码仅对信任的人开源，但会提供一个早期的机器人开源——星星0.5。采用的是当时较为落后的编程方式和实现，能力极弱，感兴趣的小伙伴可以看看。未来可能会继续开源其他版本。
> 复制、修改、再分发及其他用途以仓库中的 `LICENSE` 为准。

## 它能做什么

李尔并不是只专注于一个任务，而是拥有聊天、知识、娱乐、记忆等多种功能。以下为简要表格，仅展示了部分常用功能：

| 能力领域 | 代表功能 |
|---|---|
| 自然聊天 | 日常交流、知识问答、多轮对话、复述、澄清、发起话题 |
| 记忆 | 姓名、年龄、身份、爱好、喜好、性格、画像等|
| 常见能力 | 成语接龙、单词翻译、繁简体转换、小说等 |
| 数学能力 | 四则计算、中文数字转换、大小比较、排序筛选、简单应用题 |
| 时间与生活 | 日期、星座、定时提醒、单位换算、BMI 等 |
| 知识与推理 | 知识图谱、实体分类、属性查询、简单逻辑推理等 |

## 主要功能

### 1. 中文聊天与知识问答

- 同一问题有多种自然表达，实现了一定程度的泛化能力和回答多样性。
- 可以理解“还有吗”“继续”“接着说”“再解释一下”等连续追问。
- 当一句话包含多个问题时，李尔也可以智能处理，回答多个问题并在需要时分点作答。
- 对相似度不足但存在可能候选的问题，不直接胡乱回答，而是向用户确认意图。
- 可接受“简单说”“详细一点”等指令调整回答。

### 2. 成语能力

成语同时和内部的拼音功能和接龙功能协作，可以：

- 查询和解释成语。
- 成语接龙。

### 3. 长期记忆与个性化

李尔能够提取并管理用户主动透露的信息，包括姓名、年龄、性别、职业、身份、爱好、偏好、厌恶和性格等。

系统区分：

- **永久画像**：身份确认后的长期用户资料，可保存到本地文件。
- **临时画像**：未确认身份的当前会话信息，避免不同访客之间串线。
- **短期话题记忆**：记录最近聊天的内容，用于多轮对话、后续回复和发起话题。
- **遗忘**：用户可以要求机器人删除某一偏好或清空相关资料。

### 4. 角色扮演

不同的角色都有独立的语气和个性风格，不是简单修改文本：

- 改变句首与句尾表达。
- 改变语气词和插入词。
- 特定问题的专属回答。
- 受情绪功能影响。
- 回答长度倾向和语言风格不同。


### 5. 数学、时间与实用工具

系统内置大量确定性处理器，尽量避免把所有任务都交给模糊的知识库回答：

- 中文数字与阿拉伯数字转换。
- 四则运算、比较、排序与筛选。
- 速度、鸡兔同笼、打折等多类应用题。
- 数学练习和连续答题挑战。
- 日期、星期、时间差、季节、节日与生肖星座。
- 随机数、密码、骰子和提醒功能。
- BMI、视力等生活辅助模块。

### 6. 知识推理与主动学习

- 使用结构化知识图谱查询实体的定义、分类、特征和上下级关系。
- 支持“某物是不是某一类”“两种事物分别属于什么”等分类问题。
- 对未知实体设置守卫，避免强行给出确定结论。
- 在遇到部分未知短问题时，可以请求用户教授答案，并写入本地增量知识库。

### 7. 情绪

李尔拥有自己的情绪，受聊天内容影响。李尔所使用的表情均由当下的情绪决定。情绪还会影响会话状态，例如：当李尔心情不好时，ta可能会拒绝玩游戏或角色扮演，且可能使对话结束。

```text
   │
   ▼

   │
   ▼
终端 / Web / Android 输出
```

## 混合检索机制

李尔的知识问答并不是简单的关键词 `if` 判断。当前检索链路包括：

1. 完全一致问题快速查找。
2. 字符 n-gram 与 BM25 倒排召回。
3. 单字倒排补充候选。
4. 编辑距离、位置 n-gram 与 Jaccard 相似度。
5. 字向量语义相似度。
6. 候选集合内的精确重排。
7. 严谨、平衡、创造等不同匹配模式与阈值。
8. 其他内部精细实现。

系统只索引问题文本，回答在命中问题组后再从回答池中选择，避免长回答反向干扰问题检索。

倒排索引和问题向量支持本地缓存，并会校验知识库内容、源文件信息、参数和缓存版本，减少重复启动成本。

## 项目特色

### 完全自主的编程式路线

项目核心不是调用在线大模型，而是通过人工设计的数据、算法和状态机制完成对话。它的优势是：

- 本地运行，成本低。
- 回答来源可追踪，无幻觉。
- 行为和人格高度可控。
- 不会因为模型升级而突然改变整体风格。
- 可以针对具体中文表达逐项优化。

### 不只是“问一句、答一句”

李尔尝试把聊天本身也作为系统的一部分：

- 记住用户信息和最近话题。
- 维护机器人自己的状态。
- 不同的回答风格与上下文衔接。
- 根据上下文补充“还、也、而且、此外”等衔接表达。
- 对模糊理解进行确认。
- 在回答池耗尽时明确说明，而不是无限重复。
- 李尔拥有主动学习功能，在聊天中遇到无法回答的问题时可能会询问用户的回答并学习。此外，用户也可以直接教给李尔知识。

### 功能覆盖广

在同一个本地机器人中同时实现聊天、记忆、角色、情绪、知识图谱、数学、翻译、成语接龙、小游戏、主动学习和内容生成，是李尔最明显的特点之一。

## 运行环境

推荐使用 Python 3.10 或更高版本。

当前完整版本还依赖项目内的本地数据模块，例如：

```text
单词.py
成语.py
数据.py
```

部分功能会读取或生成：

```text
字向量_f16_pure.json       # 可选字向量数据
bot_knowledge.json         # 用户教授的增量知识
user_data.json             # 用户姓名与偏好
inverted_index_bm25.pkl    # 自动生成的 BM25 索引缓存
index_meta_bm25.json       # 索引元数据
corpus_vectors_cache.pkl   # 问题向量缓存
```

由于当前公开仓库主要用于展示和其他开源机器人版本，仓库中的旧主程序非李尔最新的全部代码和数据文件。

## 当前状态与后续计划

李尔已经形成一套功能复杂、交互细节丰富的个人中文对话系统。当前限制主要不在功能数量，而在于长期迭代后主文件规模较大、模块优先级复杂。

后续重点包括：

- 拆分主路由与大型单文件。
- 考虑加入小模型对话兜底。
- 为成语接龙、数学、记忆、翻译和连续追问建立自动化测试。
- 继续优化知识库质量和错误命中问题。
- 改善 Android 界面、会话管理与本地部署体验。
