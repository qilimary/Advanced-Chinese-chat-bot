# 李尔（Lear）

**一个完全由个人长期开发、以 Python 规则与检索系统为核心的中文多功能聊天机器人。**

李尔不是对 ChatGPT、通义千问或其他大型语言模型 API 的套壳，也不是只会进行关键词问答的简单脚本。它由中文检索、对话树、会话状态、用户记忆、角色与情绪、知识推理、实用工具、互动游戏和内容生成等模块共同组成，目标是在本地环境中实现一套**可解释、可控制、可持续扩展的编程式中文对话系统**。

> **“双鱼座”是李尔的前辈。** 它是作者更早开发的聊天系统，李尔在其代码、知识库、功能设计和对话理念上继续重构与扩展；二者属于同一条长期演进的个人中文聊天机器人路线。
>
> 当前仓库主要用于项目展示与持续开发。源码的复制、修改、再分发及其他用途以仓库中的 `LICENSE` 为准。

## 它能做什么

李尔并不是只专注于一个任务，而是把聊天机器人、个人助理、知识工具和小游戏整合进同一套离线程序中。

| 能力领域 | 代表功能 |
|---|---|
| 自然聊天 | 日常交流、知识问答、多种问法匹配、连续追问、复述、澄清、主动话题 |
| 多轮对话 | 对话树、上下文分支、复合句拆分、模糊问题确认、“还有吗”“继续说”等追问 |
| 用户记忆 | 姓名、年龄、身份、爱好、喜欢与不喜欢、性格、临时画像、永久画像、遗忘请求 |
| 角色与情绪 | 温柔、猫娘、男朋友、女朋友、高冷等角色风格，机器人心情、情绪历史与语气渲染 |
| 中文语言工具 | 成语解释、**成语接龙**、拼音、翻译、语言识别、繁简体转换、部分语法与文本处理 |
| 数学与数字 | 四则计算、中文数字转换、大小比较、排序筛选、应用题、数学挑战、随机数与密码生成 |
| 时间与生活工具 | 日期与时间差、星期、季节、节日、生肖、星座、提醒、单位换算、BMI 等 |
| 知识与推理 | 知识图谱、实体分类、属性查询、简单逻辑推理、问题相似度检索、未知知识增量学习 |
| 互动游戏 | **成语接龙、井字棋、猜数字、投骰子、知识问答、数学挑战**等 |
| 内容生成 | 故事与小说素材、诗歌、信件、求职信、简历、学习报告等模板化内容 |
| 部署与接入 | 终端运行、Web 会话接口、Android 宿主调用、多访客会话隔离与资源释放 |

### 多功能性不是简单堆积

这些能力会共享会话状态。例如，机器人可以记住用户刚才讨论的话题，在用户说“还有吗”时继续从同一回答池中补充；可以在角色扮演状态下保持对应语气；也可以在游戏、数学挑战、模糊确认或主动学习过程中等待用户下一轮输入。

## 重点功能

### 1. 中文聊天与知识问答

- 支持同一问题的多种自然表达。
- 支持回答池随机选择和已用回答去重。
- 支持“还有吗”“继续”“接着说”“再解释一下”等连续追问。
- 支持较长复合输入的拆分处理，并合并多个子问题的答案。
- 对相似度不足但存在可能候选的问题，不直接胡乱回答，而是向用户确认意图。
- 可根据“简单说”“详细一点”等指令调整回答长度。

### 2. 成语系统与成语接龙

成语不只是静态查询功能。系统包含独立的成语处理和互动状态，可以：

- 查询和解释成语。
- 识别成语拼音与读音。
- 开始多轮成语接龙。
- 检查用户输入是否为有效成语。
- 根据上一个成语的末字继续接龙。
- 维护接龙过程中的轮次和游戏状态。

### 3. 用户记忆与个性化

李尔能够提取并管理用户主动透露的信息，包括姓名、年龄、性别、职业、身份、爱好、偏好、厌恶和性格等。

系统区分：

- **永久画像**：身份确认后的长期用户资料，可保存到本地文件。
- **临时画像**：未确认身份的当前会话信息，避免不同访客之间串线。
- **短期话题记忆**：记录最近讨论的核心名词，用于后续回复和主动话题。
- **可控遗忘**：用户可以要求机器人删除某一偏好或清空相关资料。

### 4. 角色扮演与机器人情绪

角色系统并非只替换一条固定回答，而会改变：

- 句首与句尾表达。
- 语气词和插入词。
- 词语替换方式。
- 特定问题的专属回答。
- 情绪表达强度。
- 回答长度倾向和语言风格。

机器人自身还会维护一定的情绪状态，并能对重复输入、友好交流、负面表达和部分游戏请求作出不同反馈。

### 5. 数学、时间与实用工具

系统内置大量确定性处理器，尽量避免把所有任务都交给模糊的知识库回答：

- 中文数字与阿拉伯数字转换。
- 四则运算、比较、排序与筛选。
- 速度、人数、流水等多类应用题。
- 数学练习和连续答题挑战。
- 日期、星期、时间差、季节、节日与生肖星座。
- 随机数、密码、骰子和提醒功能。
- BMI、视力等生活辅助模块。

### 6. 知识推理与主动学习

- 使用结构化知识图谱查询实体的定义、分类、特征和上下级关系。
- 支持“某物是不是某一类”“两种事物分别属于什么”等分类问题。
- 对未知实体设置守卫，避免强行给出确定结论。
- 在遇到部分未知短问题时，可以请求用户教授答案，并写入本地增量知识库。

```text
用户输入
   │
   ▼
输入规范化与会话控制
   │
   ├── 特殊状态：游戏 / 提醒 / 主动学习 / 模糊确认
   ├── 用户画像：姓名 / 身份 / 偏好 / 遗忘请求
   ├── 确定性工具：数学 / 翻译 / 日期 / 单位 / 生活工具
   ├── 对话树：当前分支与根节点

   └── 通用回复与兜底机制
   │
   ▼

   │
   ▼
称呼、情绪、角色与繁简体渲染
   │
   ▼
终端 / Web / Android 输出
```

## 混合检索机制

李尔的知识问答并不是简单的关键词 `if` 判断。当前检索链路包括：

1. 完全一致问题快速查找。
2. 字符 n-gram 与 BM25 倒排召回。
3. 单字倒排补充候选。
4. 编辑距离、位置 n-gram 与 Jaccard 相似度。
5. 可选的纯 Python 字向量语义相似度。
6. 候选集合内的精确重排。
7. 严谨、平衡、创造等不同匹配模式与阈值。
8. 对话树、当前分支和知识库之间的多来源竞争。

系统只索引问题文本，回答在命中问题组后再从回答池中选择，避免长回答反向干扰问题检索。

倒排索引和问题向量支持本地缓存，并会校验知识库内容、源文件信息、参数和缓存版本，减少重复启动成本。

## 项目特色

### 完全自主的编程式路线

项目核心不是调用在线大模型，而是通过人工设计的数据、算法和状态机制完成对话。它的优势是：

- 本地运行，成本低。
- 回答来源可追踪。
- 行为和人格高度可控。
- 不会因为模型升级而突然改变整体风格。
- 可以针对具体中文表达逐项优化。

### 不只是“问一句、答一句”

李尔尝试把聊天本身也作为系统的一部分：

- 记住用户信息和最近话题。
- 维护机器人自己的状态。
- 把长回答拆成更像聊天软件的多条消息。
- 根据上下文补充“还、也、而且、此外”等衔接表达。
- 对模糊理解进行确认。
- 在回答池耗尽时明确说明，而不是无限重复。

### 功能覆盖广

在同一个本地机器人中同时实现聊天、记忆、角色、情绪、知识图谱、数学、翻译、成语接龙、小游戏、主动学习和内容生成，是李尔最明显的特点之一。

## 运行环境

推荐使用 Python 3.10 或更高版本。

当前完整版本还依赖项目内的本地数据模块，例如：

```text
主程序.py
单词.py
成语.py
数据.py
```

部分功能会读取或生成：

```text
字向量_f16_pure.json       # 可选字向量数据
bot_knowledge.json         # 用户教授的增量知识
user_data.json             # 用户姓名与偏好
inverted_index_bm25.pkl    # 自动生成的 BM25 索引缓存
index_meta_bm25.json       # 索引元数据
corpus_vectors_cache.pkl   # 问题向量缓存
```

由于当前公开仓库主要用于展示和历史版本存档，仓库中的旧主程序未必包含李尔最新的全部代码和数据文件。

## 最近修复：连续追问只输出表情

旧版“还有吗”模板中包含空字符串。当随机命中空模板时，已经取得的补充正文会被丢弃，而情绪层仍可能附加 emoji，最终界面只显示一朵花或一个表情。

修复内容包括：

- 空模板改为能够保留正文的 `"{}"`。
- 模板格式化后增加空正文回退。
- 情绪系统禁止在没有正文时单独生成 emoji。
- 回答池过滤空字符串和纯空白内容。
- 回答耗尽后保持耗尽状态，不再立即从头循环。

更完整的结构与技术分析见 [TECHNICAL_REPORT.md](TECHNICAL_REPORT.md)。

## 当前状态与后续计划

李尔已经形成一套功能复杂、交互细节丰富的个人中文对话系统。当前限制主要不在功能数量，而在于长期迭代后主文件规模较大、模块优先级复杂。

后续重点包括：

- 拆分主路由与大型单文件。
- 建立统一的意图路由和处理器返回协议。
- 为成语接龙、数学、记忆、翻译和连续追问建立自动化测试。
- 继续优化知识库质量和错误命中问题。
- 改善 Android 界面、会话管理与本地部署体验。
